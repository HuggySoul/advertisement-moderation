# Инструкция по развёртыванию с Docker

- **ВАЖНО!** Клиент и сервер в проекте являются git-подмодулями. Поэтому для клонирования репозитория с подмодулями выполните:
  ```
  git clone --recurse-submodules https://github.com/HuggySoul/advertisement-moderation.git
  ```
- Запустите Docker или установите с [официального сайта](https://www.docker.com/products/docker-desktop/)
- Выполните команду для сборки проекта и запуска контейнеров в корневой папке проекта

  ```sh
  docker-compose up --build
  ```

- Перейдите по адресу http://localhost:8080/ для просмотра клиентской части
- Сервер работает по адресу http://localhost:3001/api/v1
- [Более подробно о конфигурации сервера](https://github.com/HuggySoul/ad-moderation-server.git)

# Инструкция по развёртыванию с npm

### Запуск сервера

- **ВАЖНО!** Клиент и сервер в проекте являются git-подмодулями. Поэтому для клонирования репозитория с подмодулями выполните:

  ```
  git clone --recurse-submodules https://github.com/HuggySoul/advertisement-moderation.git
  ```

- Перейдите в директорию `./ad-moderation-server` и выполните команду для установки node-модулей:

  ```sh
  npm i
  ```

- В этой же директории для запуска выполните:

  ```sh
  npm start
  ```

- Сервер работает по адресу http://localhost:3001/api/v1

### Запуск клиента

- Перейдите в директорию `./ad-moderation-client` и выполните команду для установки node-модулей:

  ```sh
  npm i
  ```

- В этой же директории для запуска в режиме разработки:

  ```sh
  npm run dev
  ```

- Перейдите по адресу `http://localhost:8080/`

# Клиент

Структура построена по принципу FSD, но с некоторыми упрощениями из-за небольшого размера проекта.

FSD выбрал по так как с ним имею больший опыт взаимодействия, по сравнению с другими подходами.

### Используемые технологии и обоснование

- Vite для сборки проекта. Очень прост в настройке, что актуально в условиях ограниченного времени.
- Redux Tool Kit. В частности использую RTK Query в качестве серверного стейт-менеджера. Выбрал из-за удобной автогенерации хуков, а также для того, чтобы использовать в связке с Redux, однако по итогу Redux не потребовался из-за простоты проекта.
- Ant-design. Решил воспользоваться готовой дизайн-системой, чтобы сократить время на вёрстку. Попробовал её впервые: поначалу было непривычно и неудобно, но последствии библиотеки ощутимо ускорили процесс создания макета.
- ИИ использовался для ускорения разработки конфига Ant-design, а также ускорения поиска информации о библиотеках Ant-design.
- prettier(в проекте нет, использовал как расширение). Давно его использую для форматирования кода, удобен, относительно гибок в настройке.
